# Support

## Scope

Veeling is maintained as an open-source project with best-effort support.

Support is focused on:

- Reproducible bugs
- Installation and upgrade questions for officially documented channels
- Clarifications about documented behavior

Support is generally **not** guaranteed for:

- Custom forks and heavily modified environments
- End-of-life or very old pre-1.0 releases
- Undocumented internal APIs or implementation details

## How to get help

1. Read `README.md` and `CONTRIBUTING.md` first.
2. Search existing GitHub issues and pull requests.
3. Open a new GitHub issue using the most appropriate template.

## Response expectations

- Maintainers triage issues as time permits.
- High-quality reports with clear reproduction steps are prioritized.
- Security concerns should follow `SECURITY.md` and must not be filed publicly.

## Issue quality checklist

When opening an issue, include:

- Veeling version
- OS and runtime details
- Exact command(s) run
- Expected vs actual behavior
- Minimal reproduction where possible
