# Security Policy

## Supported Versions

Veeling is pre-1.0 and follows a latest-release support model.

| Version | Supported |
| :-- | :-- |
| Latest `0.x` release | ✅ |
| Older `0.x` releases | ⚠️ Best effort only |
| `main` (unreleased) | ❌ |

## Reporting a Vulnerability

Please **do not** open public GitHub issues for security vulnerabilities.

Use one of these private channels:

1. Preferred: GitHub Security Advisory reporting for this repository.
2. Fallback: contact maintainers privately through the repository owner contact listed in the repository profile.

When reporting, include:

- Affected version(s)
- Reproduction steps or proof-of-concept
- Impact and attack prerequisites
- Any suggested mitigation

## Response Targets

- Initial acknowledgement: within **3 business days**
- Triage decision (accepted/rejected/needs more info): within **7 business days**
- Coordinated fix/release target for accepted issues: as soon as feasible, prioritized by impact

## Disclosure Process

- We aim for coordinated disclosure.
- We will credit reporters unless anonymity is requested.
- Once a fix ships, release notes will include a security section where applicable.
